HAPI 2.0 server verifier. See verifier-nodejs for source code for tests.

Usage: ?url=URL&id=ID&parameter=PARAMETER&time.min=ISO8601&time.max=ISO8601.

Alternative usage: ?url=http://server/hapi?id=ID&parameter=PARAMETER&time.min=ISO8601&time.max=ISO8601.

Only a URL that ends in /hapi is required. Only one ID and one PARAMETER may be specified (comma-separated list not supported).

Only URL is required. Default is to check all datasets and all parameters and use time.min=sampleStartDate and time.max=sampleStopDate if both given in /info response, otherwise the required time.min=startDate and time.max=startDate+P1D from /info are used.

A list of things that are not tested are given in the TODO section for the project README.

Usage Examples:

  • Test everything: ?url=http://mag.gmu.edu/TestData/hapi
  • Test only dataset1: ?url=http://mag.gmu.edu/TestData/hapi&id=dataset1
  • Test parameter scalar in dataset1: ?url=http://mag.gmu.edu/TestData/hapi&id=dataset1&parameter=scalar
  • Test parameter scalar in dataset1 in a given timerange: ?url=http://mag.gmu.edu/TestData/hapi&id=dataset1&parameter=scalar&time.min=1970-01-01&time.max=1970-01-01T00:00:10
